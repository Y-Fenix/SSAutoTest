# SSAutoTest 另一台 Mac 一键部署

把本文件交给另一台 Mac 上的 Codex，让它按下面步骤执行即可。

## 给 Codex 的执行提示词

```text
请在这台 macOS 上一键部署 SSAutoTest 局域网常驻版本，目标是让我和同事在同一个 WiFi 下通过浏览器访问使用，并支持几个 G 的 CSV/TXT 实际数据文件上传后由服务器流式解析。

请按以下步骤执行：

1. 准备项目代码
   - 如果当前目录已有 SSAutoTest 项目，进入该目录。
   - 如果没有，请执行：
     git clone https://github.com/Y-Fenix/SSAutoTest.git
     cd SSAutoTest

2. 检查运行环境
   - 确认 macOS 可用。
   - 执行 node -v，要求 Node.js 20+。
   - 执行 npm -v，确认 npm 可用。
   - 如果没有 Node.js 或版本过低，请先提示我安装 Node.js 20+，不要继续硬装。

3. 安装、测试、构建
   - npm ci
   - npm test
   - npm run build

4. 安装局域网常驻服务
   - chmod +x ./一键启动局域网版本.command ./停止局域网常驻服务.command ./scripts/start-lan-service.sh
   - printf '\n' | ./一键启动局域网版本.command

5. 验证服务
   - curl -I http://localhost:5173/
   - 获取本机 WiFi IP：
     ipconfig getifaddr en0 || ipconfig getifaddr en1
   - 告诉我同事访问地址，格式为：
     http://本机局域网IP:5173/

6. 验证大文件链路
   - 说明“实际 SQL 导出 CSV/Excel”模块中，CSV/TXT 会上传到这台 Mac 的 Node 服务端做流式扫描。
   - 几个 G 的 CSV/TXT 不会在浏览器内完整解析成内存对象；页面会显示上传/扫描进度。
   - Excel 仍走浏览器读取，不建议上传几个 G 的 Excel；大文件请从数数导出 CSV。

7. 告诉我运维信息
   - 常驻服务标签：com.yfenix.ssautotest.lan
   - 日志目录：~/Library/Logs/SSAutoTest/
   - 停止服务：双击 停止局域网常驻服务.command
   - 重启服务：
     launchctl kickstart -k gui/$(id -u)/com.yfenix.ssautotest.lan
```

## 使用说明

1. 服务器 Mac 和同事电脑必须在同一个 WiFi 或同一个局域网。
2. 服务器 Mac 需要保持开机并登录当前 macOS 用户。
3. 同事打开 `http://服务器IP:5173/`，就可以像本机一样上传文件和操作页面。
4. 大 CSV/TXT 上传后由服务器 Mac 解析；同事浏览器只负责上传和显示结果。
5. 飞书表格读取使用服务器 Mac 上的 `lark-cli` 登录态。
6. 数数查询使用服务器 Mac 本地配置里的 API Token / WebSocket 配置。

## 大文件能力边界

- 推荐：CSV / TXT，几 GB 文件走服务端流式扫描。
- 不推荐：几 GB Excel。Excel 需要浏览器或 xlsx 库展开工作簿，内存压力很高。
- 大 CSV 扫描会保留覆盖分析所需摘要、事件计数、属性覆盖和有限属性值样本，不会保存全部原始行。
- 顺序类“测试用例”需要原始事件行；大 CSV 摘要扫描不保留完整原始行，因此不适合跑顺序逻辑测试。

## 常用命令

```bash
# 进入项目
cd SSAutoTest

# 安装依赖
npm ci

# 测试和构建
npm test
npm run build

# 安装并启动常驻服务
printf '\n' | ./一键启动局域网版本.command

# 重启常驻服务
launchctl kickstart -k gui/$(id -u)/com.yfenix.ssautotest.lan

# 查看服务
launchctl print gui/$(id -u)/com.yfenix.ssautotest.lan

# 查看日志
tail -n 200 ~/Library/Logs/SSAutoTest/lan-service.out.log
tail -n 200 ~/Library/Logs/SSAutoTest/lan-service.err.log

# 停止服务
./停止局域网常驻服务.command
```

## 排查

- 同事打不开页面：检查服务器 Mac 和同事是否同 WiFi，检查 `http://localhost:5173/` 是否能在服务器本机打开。
- 访问被拦截：macOS 弹出网络权限时允许 Node/Vite 接收局域网连接。
- 飞书读取失败：在服务器 Mac 上执行 `lark-cli auth login` 并确认有表格权限。
- 大文件没有进度：刷新页面后重新上传；同时查看 `~/Library/Logs/SSAutoTest/lan-service.err.log`。
- 端口占用：检查是否已有 5173 服务，必要时停止旧服务后重新执行一键启动。
