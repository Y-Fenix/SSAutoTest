# SSAutoTest

埋点覆盖核对工具，用于读取飞书预期埋点表和实际 SQL 导出 CSV/Excel，对比事件、属性覆盖情况，并输出可视化明细。

## 功能

- 读取飞书表格页签，默认选择前 3 个页签。
- 上传实际 SQL 导出 CSV/Excel。
- 自动识别 `#event_name` 事件名列。
- 检查事件缺失、属性缺失、公共事件属性缺失。
- 统计每个事件触发次数。
- 支持导出核对结果 CSV。
- 支持局域网访问，方便团队共享。
- 支持同事通过浏览器上传大 CSV/TXT，由服务器 Mac 流式扫描，适合几 GB 的数数导出文件。
- 实际数据 CSV/TXT 扫描走服务器队列，同一时间只处理 1 个大文件，避免多人同时上传导致服务器内存和 IO 被打满。
- 测试用例逻辑独立在 `public/event-test-cases.js`，替换该文件并刷新页面即可更新规则。

## 本机启动

```bash
npm ci
npm run dev
```

## 局域网常驻启动

macOS 上双击：

```text
一键启动局域网版本.command
```

它会安装用户级常驻服务：

- 关闭终端窗口后服务仍会继续运行。
- 登录 macOS 后自动启动。
- 局域网访问地址会在启动窗口里显示。

说明：这是 macOS 用户级常驻服务，会在当前用户登录后自动启动；电脑开机但未登录时不会运行。

同事在同一 WiFi 下访问局域网地址即可使用。实际 SQL 导出的 CSV/TXT 会先进入服务器扫描队列，轮到该任务时才开始上传/解析；Node 服务端边接收边解析，同一时间只跑 1 个大文件任务。Excel 仍走页面读取，不建议上传几 GB 的 Excel，大文件请优先导出 CSV。

停止常驻服务：

```text
停止局域网常驻服务.command
```

## 临时局域网启动

```bash
npm ci
npm run build
npm run preview:lan
```

启动后终端会输出：

```text
Network: http://局域网IP:5173/
```

同事在同一局域网内访问该地址即可。

## 飞书读取

飞书链接读取依赖当前电脑的 `lark-cli` 登录态。

```bash
lark-cli auth login
```

如果读取 Wiki/表格时报权限问题，请按页面提示补充对应 scope。
